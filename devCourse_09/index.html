프로그래머스 데브코스 과정 중 9월 한달간의 회고

📝 9월 회고

들어가면서,

9월에는 9월 1일(금) ~ 9월 27일(수) 약 4주간의 팀프로젝트를 진행하였다.
팀은 프론트엔드 개발자 5명으로 이루어져 있고 백엔드는 강사님이 관리해 주셨다.
프로젝트내용은 혹시 몰라서 자세하게는 작성하지 않으려 한다.
기술에대한 이야기 보다는 솔직한 나의 생각을 적어보려고 한다.

우리 팀은 분업을 페이지단위로 진행하였다.
그중에서 내가 맡게 된 페이지는 회원가입, 로그인, 비밀번호 변경, 알림 페이지였다.
나는 인증, 인가에 대한 관심이 있었고 전부터 구현해 보고 싶은 기능이었기에 각자 담당할 페이지를 정할 때 회원가입, 로그인 페이지를 맡았다. 추가적으로 GitHub Wiki도 작성했다.

개발자로 학습/취준을 시작하면서 처음으로 팀 프로젝트를 진행해 보았다.
이번 프로젝트를 통해 협업의 중요성을 느꼈고, 협업을 하면서 부족한 부분을 많이 느꼈다.
프로젝트를 진행하면서 느낀 점과 아쉬웠던 점, 그리고 앞으로 개선해 나갈 점들을 정리해 보았다.

👍 Keep

모르는 것은 물어보자!
나는 모르는 것은 잘 안 물어보고 남에게 도움을 잘 청하지 않고 혼자 해결하려고 하는 성격이다.
이번 프로젝트를 진행하면서는 내 마음대로 했다가는 팀에 피해를 줄 수 있기 때문에 모르는 것은 사소한 것이라도 물어보자는 생각으로 진행했다.
팀 분위기가 너무 좋았고 좋은 팀원들을 만나서 너무 친절하게 알려줘서 덕분에 이제는 질문하는 것에 대한 두려움이 조금은 사라진 것 같다.

🚨 Problem

가장 아쉬운 부분은 내 할 일에만 너무 갇혀있었던 부분인 것 같다.
좋게 생각한다면 내가 맡은 페이지에 대해서는 빠르게 구현하기 위해 집중하고 노력했다고 생각한다.
하지만 팀원들이 어떤 식으로 작업을 진행하고 있는지, 어떤 식으로 작업을 진행하고 있는지에 대해 관심을 갖지 못했다.
팀에서 PR리뷰를 진행하긴 했지만 더 꼼꼼히 볼 필요가 있다고 느낀다.
팀에게 도움이 되고 싶었고 내가 맡은 페이지는 정해진 기간 안에 무조건 완성해야 한다는 생각에 조금함과 불안함이 있었고 그생각에 갇혀있었다고 생각한다.
그래서 프로젝트의 전체적인 흐름을 파악하지 못했고, 내가 맡은 페이지에 대해서만 집중하고 있었다.
이번 프로젝트를 진행하면서 처음 사용해 보는 라이브러리와 기술들이 많았는데, 그것들을 깊게 공부할 시간이 되지 않았다.
그래서 일단 내가 아는 지식으로 돌아가는 코드로 만들어서 빠르게 구현하고, 나중에 시간이 되면 다시 공부하고 개선해 나가야겠다고 생각했다.

✨ Try

다음 프로젝트는 조금 더 넓은 시야로 프로젝트를 보고 진행하고 싶은 마음이 든다.
그러기 위해서는 내가 맡은 페이지에 대해서만 갇혀있지 않고 전체적인 흐름을 파악하고, 팀원들이 어떤 식으로 작업을 진행하고 있는지, 어떤식으로 작업을 진행하고 있는지에 대해 관심을 가져야 할 것이다.
그것을 실천하기 위해서는 PR리뷰를 더 꼼꼼히 해야 할 것 같다.
조급함을 버리자. 나를 조금 더 믿어주자.

마지막으로,

팀원들에게 많은 도움을 받은 것 같아서 고마운 마음이 많이 든다.
표현은 하지 못했지만, 팀 프로젝트를 하면서 우리 팀원들에게 고마운 마음을 많이 느꼈습니다.
부족한 점이 많았는데 그럼에도 불구하고 저를 믿어주고, 도와주고, 친절하게 알려주셔서 너무 감사했습니다.
